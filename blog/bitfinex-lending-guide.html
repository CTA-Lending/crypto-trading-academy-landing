Bitfinex 放貸完整教學：從開戶到第一筆自動放貸

2026 年 4 月 8 日 | 閱讀時間約 8 分鐘

什麼是 Bitfinex 放貸？

如果你持有加密貨幣但不想頻繁交易，融資放貸（Funding）是一種讓閒置資金產生收益的方式。簡單來說，Bitfinex 交易所上有一群使用槓桿交易的用戶，他們需要借入資金來開倉。而你，就是把錢借給這些交易者的人。

整個過程由 Bitfinex 平台撮合，你不需要知道錢借給了誰，也不用擔心對方不還錢。平台會透過保證金機制確保借款人在虧損到一定程度時自動平倉，從而保護放貸方的本金安全。

你可以放貸的幣種包括 USD、UST（USDT）、BTC、ETH 等主流資產。其中以 USD 和 UST 的放貸需求最大、流動性最好。放貸利率由市場供需決定，當市場交易活躍、槓桿需求高時，利率會明顯上升；反之則下降。

核心概念

放貸本質上是「你把穩定幣借給做槓桿交易的人，賺取利息」。Bitfinex 作為中間人撮合交易，並透過保證金制度保護你的資金。

誰適合使用放貸？

放貸適合以下幾種人：

  • 持有穩定幣但暫時不交易的人 — 與其讓 USDT 閒置在錢包裡，不如讓它每天產生利息。
  • 追求被動收入的投資者 — 放貸不需要盯盤、不需要判斷行情方向，是真正的「設定好就忘記」。
  • 不想承受幣價波動風險的人 — 放貸穩定幣（如 USDT）的本金不會因為幣價漲跌而改變。
  • 想用自動化工具提升效率的人 — 搭配放貸機器人，可以 24 小時不間斷地在最佳利率掛單。

當然，放貸並非完全零風險（後面會詳細說明），但對於想要穩健增值的人來說，它是加密貨幣世界中風險相對較低的一種策略。

開始前的準備

第一步：註冊 Bitfinex 帳號

前往 Bitfinex 官網註冊帳號。註冊過程需要提供電子信箱並設定密碼。建議開啟兩步驟驗證（2FA）以確保帳號安全。

第二步：完成 KYC 身分驗證

Bitfinex 要求用戶完成身分驗證才能使用完整功能。你需要準備：

  1. 有效的身分證件（護照或身分證）
  2. 地址證明文件（如銀行對帳單或水電帳單，三個月內）
  3. 自拍照片（手持身分證件）

審核時間通常在數天到兩週之間。完成 KYC 後即可進行法幣入金或加密貨幣充值。

第三步：將資金轉入 Funding Wallet

Bitfinex 帳戶內有三個錢包：Exchange（交易）、Margin（保證金）和 Funding（融資）。放貸必須使用 Funding Wallet，所以你需要在帳戶內將資金從其他錢包轉移到 Funding Wallet，這是站內即時轉帳，不需手續費。

第四步：API Key 設定（使用機器人時需要）

如果你打算使用 CTA 放貸機器人或其他自動化工具，需要建立 API Key：

  1. 進入 Bitfinex 的 API 管理頁面
  2. 建立新的 API Key
  3. 權限設定：只需開啟 「Funding」相關的讀取與寫入權限，不要開啟提款權限
  4. 保存好 API Key 和 Secret — Secret 只會顯示一次

安全提醒

建立 API Key 時，絕對不要開啟「提款」權限。只開放 Funding 相關權限，即使 API Key 外洩，你的資金也不會被轉走。

手動放貸 vs 自動放貸

了解完基本設定後，接下來面臨一個選擇：自己手動掛單，還是用機器人自動處理？

比較項目 手動放貸 自動放貸（機器人）
操作方式 手動在 Bitfinex 掛單 API 連接，全自動執行
掛單時間 只有你在線時 24 小時不間斷
利率最佳化 依個人經驗判斷 演算法即時分析最佳利率
資金利用率 經常有閒置資金 到期即重新掛單，利用率高
適合對象 小額測試、想了解機制 長期放貸、追求效率
每日花費時間 15-30 分鐘 幾乎為零

手動放貸的最大問題在於時間成本和資金閒置。每次放貸到期後，你需要手動重新掛單。如果你在睡覺或工作，資金就只能空轉。而自動放貸機器人會在到期的第一時間重新掛單，大幅提升資金利用率。

CTA 放貸機器人如何運作

交易幣修課（CTA）開發的放貸機器人是一套 SaaS 工具，透過 API 連接你的 Bitfinex 帳戶，全自動管理放貸流程。以下是六大核心功能：

1. 智慧利率演算法

機器人會即時分析市場上的放貸訂單簿（Order Book），結合歷史利率數據，自動計算當下最佳的掛單利率。不會掛太高導致成交不了，也不會掛太低讓你少賺。

2. 自動續貸

每筆放貸到期後，機器人在數秒內自動重新掛單，確保你的資金幾乎零閒置時間。這是手動操作很難做到的。

3. 分散到期策略

將資金分成多筆、不同天期的放貸單，避免所有資金同時到期造成利率風險集中。類似債券投資中的「階梯策略」。

4. 最低利率保護

你可以設定最低可接受利率，當市場利率低於門檻時，機器人會暫停掛單，等利率回升再出手，避免在低利率時段鎖定資金。

5. 即時監控儀表板

透過網頁後台即時查看目前的放貸狀態、累計收益、資金利用率、歷史利率曲線等關鍵數據，隨時掌握資金動態。

6. 多幣種支援

支援 USD、UST（USDT）等主流放貸幣種，可同時管理多個幣種的放貸，一個帳號統一操作。

風險與注意事項

雖然放貸相較於交易風險較低，但並非完全沒有風險。在開始之前，你應該了解以下幾點：

平台風險

你的資金存放在 Bitfinex 交易所上，如果交易所發生安全事件（如駭客攻擊、系統故障），你的資金可能受到影響。建議不要將所有資產都放在同一個平台上，做好資金分散配置。

利率波動風險

放貸利率由市場供需決定，沒有固定報酬。在市場冷淡時期，利率可能會低到接近零。不要把放貸收益當作保證的固定收入來做財務規劃。

流動性鎖定風險

放貸單在到期前無法提前取回。如果你掛了一筆 30 天的放貸單，這段期間資金就被鎖定。如果你可能需要隨時動用資金，建議選擇較短天期（如 2-7 天）的放貸。

極端行情風險

在極端市場波動下（如閃崩），借款人的保證金可能不足以覆蓋損失。雖然 Bitfinex 有自動平倉機制和保險基金，但在極端情況下仍有微小的損失可能。歷史上這種情況非常罕見，但並非不可能發生。

風險管理建議

  • 不要將全部資產投入放貸，保留足夠的流動資金
  • 優先選擇主流幣種（USD、UST）進行放貸
  • 設定合理的最低利率門檻，不要為了追求成交率而接受過低利率
  • 定期檢視放貸收益和市場狀況

常見問題

Q：放貸的利息多久發放一次？

利息在每筆放貸單到期時結算。例如你掛了一筆 2 天的放貸單，到期後利息就會自動入帳到你的 Funding Wallet。如果使用自動放貸機器人，到期的本金加利息會立即被重新掛單出去，產生複利效果。

Q：最少需要多少資金才能開始放貸？

Bitfinex 的最低放貸金額約為 50 USD（或等值加密貨幣）。不過建議至少準備 500-1000 USD 以上，這樣機器人可以更靈活地分散掛單，利率最佳化的效果也更明顯。

Q：放貸需要繳手續費嗎？

Bitfinex 會從放貸收益中收取 15% 作為平台手續費（直接從利息中扣除）。也就是說，如果你賺到 100 元利息，實際入帳是 85 元。使用 CTA 放貸機器人則有額外的 SaaS 服務費用，詳情請參考我們的方案頁面。

Q：使用 CTA 放貸機器人安全嗎？

CTA 機器人僅透過 API 執行放貸操作，你的資金始終在你自己的 Bitfinex 帳戶中。只要你在建立 API Key 時不開放提款權限，機器人就無法將你的資金轉出。你隨時可以在 Bitfinex 後台撤銷 API Key 以中止機器人授權。
